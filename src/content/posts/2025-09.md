### Logback 日志框架配置警告

::: details 弃用的 `<level>` 元素
    WARN in ch.qos.logback.classic.joran.action.LevelAction - <level> element is deprecated. Near [level] on line 55
    Please use "level" attribute within <logger> or <root> elements instead.

**旧写法 (已废弃)**
```xml
<logger name="com.example.myapp">
    <level>DEBUG</level> <!-- 就是这行导致的警告 -->
    <appender-ref ref="CONSOLE"/>
</logger>

<root>
    <level>INFO</level> <!-- 在root里这么写也会警告 -->
    <appender-ref ref="FILE"/>
</root>
```

**新写法 (推荐) 改用 level 属性**
```xml
<logger name="com.example.myapp" level="DEBUG"> <!-- 使用 level 属性 -->
    <appender-ref ref="CONSOLE"/>
</logger>

<root level="INFO"> <!-- 在root元素上也使用 level 属性 -->
    <appender-ref ref="FILE"/>
</root>
```
:::

::: details 弃用的 SizeAndTimeBasedFNATP 策略

    SizeAndTimeBasedFNATP is deprecated. Please use SizeAndTimeBasedRollingPolicy instead.

**旧写法 (已废弃)**
```xml
<appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>app.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
        <fileNamePattern>app.%d{yyyy-MM-dd}.%i.log.gz</fileNamePattern>
        <timeBasedFileNamingAndTriggeringPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedFNATP">
            <maxFileSize>100MB</maxFileSize>
        </timeBasedFileNamingAndTriggeringPolicy>
        <maxHistory>30</maxHistory>
    </rollingPolicy>
    ...
</appender>
```
**新写法 (推荐) SizeAndTimeBasedRollingPolicy，这个新策略将滚动和触发策略合二为一，配置更简洁**
```xml
<appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>app.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
        <!-- 核心变化：直接使用这个Policy -->
        <fileNamePattern>app.%d{yyyy-MM-dd}.%i.log.gz</fileNamePattern>
        <!-- maxFileSize 现在直接是 rollingPolicy 的一个属性 -->
        <maxFileSize>100MB</maxFileSize>
        <maxHistory>30</maxHistory>
        <!-- 你还可以设置总大小限制 -->
        <totalSizeCap>3GB</totalSizeCap>
    </rollingPolicy>
    ...
</appender>
```

| -XX:MaxMetaspaceSize | 最大元空间大小           | 128 MB    | 256 MB    | 512 MB    | 512 MB     |
| -Xss                 | 每个线程可使用的内存大小  |           |           |           |            |

JVM 使用内存 ≈ Xmx + MaxMetaspaceSize + 线程数 x Xss
